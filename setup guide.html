LAK-Market Place

A comprehensive setup guide for the Lakshadweep Marketplace project, covering backend, frontend, admin panel, and mobile app components.

1. Prerequisites

Ensure you have the following installed on your system:

  • Node.js: v18 or higher (v20+ recommended)
  • npm: v9 or higher
  • MySQL: v8.0
  • Flutter SDK: v3.0 or higher
  • Docker: Optional, for running the MySQL container

2. Backend Setup (Node.js/Express)

Environment Configuration

Create a .env file in the backend/ directory with the following integration credentials:

# Server
PORT=5000
NODE_ENV=development

# Database (MySQL)
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=rootpassword
DB_NAME=lak_marketplace

# Auth & Security
JWT_SECRET=your_jwt_secret_key
OTP_EXPIRY_MINUTES=5

# Email (Nodemailer/Gmail)
EMAIL_USER=your-email@gmail.com
EMAIL_PASS=your-app-specific-password

# Marketplace Logic
COMMISSION_PERCENTAGE=5
Security Note: For Gmail integration, ensure you use an App Password. Direct account passwords will be blocked by Google's security policy.

Database Initialization

Run the backend/database_schema.sql script in your MySQL instance. Alternatively, use the provided Docker configuration:

docker-compose up -d

Installation & Execution

cd backend
npm install
npm run dev

3. Frontend Setup (React/Vite)

The frontend is built with React and Tailwind CSS. Update your frontend/.env for local development:

VITE_API_URL=http://localhost:5000/api
cd frontend
npm install
npm run dev

4. Mobile App Setup (Flutter)

The mobile application uses Flutter. Ensure your developmental device can reach the backend server.

Configure the baseUrl in mobile-app/lib/services/api_service.dart:

static const String baseUrl = 'http://YOUR_SERVER_IP:5000/api';
cd mobile-app
flutter pub get
flutter run

6. Cloud Sync & Deployment Flow

Automated Synchronization

This project uses an Asynchronous Cloud Synchronization workflow:

  • Source Control: GitHub Repository linked to haneefa0007-cyber.
  • CI/CD Bridge: Pushing changes to the main branch automatically triggers secondary builds on Render and Vercel.
  • Backend Hosting: Render handles Express server and Socket.io scaling.
  • Frontend Hosting: Vercel provides edge-optimized delivery for the React application.
